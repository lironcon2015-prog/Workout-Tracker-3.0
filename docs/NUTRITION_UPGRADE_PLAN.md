## A2 — טבעות אדהרנס עם קידוד צבע 🟢 *(גם "טבעות אדהרנס" מהכיוונים הפתוחים ב-ROADMAP)*

**מה:** הטבעות והפסים היומיים מקבלים מצב צבע סמנטי (חסר / בטווח / חריגה) לפי % מהיעד + פס שבועי של 7 נקודות.

**מצב קיים:** טבעת קלוריות `_fdRingSVG` (`food-logic.js`, ~שורה 655) עם class `over` בחריגה; פסי מאקרו `_fdMacroStat` (~670) ניטרליים; יתרה שבועית `_fdWeeklyBalance` (~578) כבר עושה לולאת שבוע עם `getTargetsForDate` פר-יום.

**שינויים (`food-logic.js` + `style.css`):**
1. **טבעת קלוריות** — מצב `good` (ירוק, `--success`): צריכה בטווח 90%–105% מהיעד; `over` (אדום, קיים) יחול מעל 105% במקום מעל 100%. מתחת ל-90% — המראה הנוכחי (accent gradient).
2. **פסי מאקרו** — חלבון: class `met` (ירוק) כשהערך ≥ היעד; פחמימה/שומן: class `over` (אדום) מעל 110% מהיעד. מימוש ב-`_fdMacroStat` (פרמטר `mode: 'floor'|'ceil'`) + CSS.
3. **פס שבועי** — שורת 7 נקודות (א׳–ש׳) מתחת לסיכום היומי ב-`_fdSummaryHTML` (~704): ירוק = היום עמד ביעד (קלוריות ±10% וגם חלבון ≥90%), אדום = תועד אך פספס, אפור/ריק = אין תיעוד/יעד/עתיד. פונקציה חדשה `_fdWeekDotsHTML(dateStr)` שמשכפלת את תבנית הלולאה של `_fdWeeklyBalance` (byDate + `getTargetsForDate`). לחיצה על נקודה מנווטת לאותו יום (`_fdDate = d; fdRender()`), היום המוצג מסומן.
4. CSS חדש: `.fd-ring-prog.good`, `.fd-mline.met/.over`, `.fd-week-dots` — צבעים סמנטיים קיימים בלבד (`--success`/`--danger`).

**לא נוגעים:** לוגיקת המספר המרכזי / יתרה שבועית ממוזגת (`_fdWkbalMerged`) — נשארת כמות שהיא.

## A3 — חיזוק אמינות מקורות בחיפוש 🟢

**תגלית מהמיפוי:** דירוג בשכבות כבר קיים ב-`fdDoSearch` (`food-logic.js` ~928–968, מ-v16.88): שלך → מועדפים → חומרי גלם → רשת מדורגת (`srcPri`: צמ"ת>OFF>USDA, ~959) → cache. **אין לבנות דירוג מחדש — רק להשלים שני פערים:**
1. **תגית "מאומת"** — ב-`_fdSrcChip` (~1075): סימון ✓ קטן (class `verified`) למקורות צמ"ת/USDA/OFF-עם-ברקוד/בסיסי; מקורות `gemini` (AI) — בלי סימון. CSS קטן ל-chip.
2. **תיעדוף בתוך שכבת ה-cache** — ב-`cacheStrong` (~943): שובר-שוויון לפי אמינות מקור (מאומת לפני `gemini`) אחרי ציון הרלוונטיות, באותו סגנון `srcPri`.

## A1 — סיבים, סוכר, נתרן 🟡

**עיקרון:** שדות אופציונליים (nullable) — מזון בלי נתונים מציג "—", לא 0. **נתרן נשמר במ"ג.**

### `food-logic.js`
1. **מיפוי OFF** — `_offToFood` (~89–108): הוספת `fiber_100g`, `sugars_100g`, `sodium_100g` (גרם→מ"ג ×1000) ל-`per100` כ-`{fb, sg, na}` (null אם חסר).
2. **סריקת תווית** — `_callGeminiFood` (~431): הוספת `fiber, sugar, sodium_mg` לסכמה + maxTokens 200→280; הצרכן (~1045, בניית food) ממפה אותם.
3. **עורך מזון מותאם** — `_fdShowCustomFoodForm`: שורה מתקפלת "ערכים נוספים (רשות)" עם 3 שדות; `fdSaveCustomFood`: אותו scale שחל על המאקרו (×100 ליחידה ללא משקל, ×k ליחידה עם משקל) חל גם עליהם, null-aware.
4. **חישוב רשומה** — `_fdMacrosFor` (~1213) מרחיב ל-fb/sg/na (null-aware); `fdSavePortion` שומר אותם על הרשומה.
5. **סיכום יומי** — `fdRender` (~625): ה-reduce סוכם גם fb/sg/na (רק ערכים שאינם null; אם אין אף נתון — null); `_fdSummaryHTML`: שורת chips משנית מתחת ל-`.fd-macros`: `סיבים 18/30 · סוכר 42/50 · נתרן 1,900/2,300` — סיבים ירוק מעל היעד (מינימום), סוכר/נתרן אדום מעל (מקסימום). ימים בבעלות MFP או ללא נתונים — השורה מוסתרת.
6. **מנה מצולמת / Meal Builder — מחוץ לסקופ** (רכיבי composite נשארים 4 שדות; רשומות כאלה יקבלו null) — חוב טכני מתועד.

### `storage.js`
7. **יעדים** — ברירות מחדל חדשות ב-analytics prefs (~230–240): `fiberTarget:30`, `sugarLimit:50`, `sodiumLimit:2300`. החלטת מימוש: נקראים ישירות מה-prefs, **בלי** מנגנון ההיסטוריה `recordTargetChange` — אלו יעדי בריאות יציבים, לא יעדי דיאטה משתנים.
8. **סכימת היום** — `recomputeNutritionDay` (~1082): הוספת `fiber/sugar/sodium` לרשומת היום (null-aware) — זמין לגרפים עתידיים ולסנכרון.
9. **עריכת יעדים** — הוספת 3 השדות ל-UI הקיים של עריכת יעדי מאקרו (לאתר קריאות `saveMacroTarget`/`saveKcalTarget` בזמן מימוש).

### תאימות לאחור
מזונות/רשומות ישנים בלי השדות — null בכל השרשרת; אין מיגרציה. הסנכרון לענן (config) סוחב את השדות אוטומטית.

## נספח — מסלולים שלא נבחרו (לתיעוד, שלא יאבדו)

### מסלול B — אימון ו-Accountability (בהשראת Trainerize)
- **B1. תכנות לפי %1RM (Auto-Progression)** 🔴 — "3×5 @ 80%" עם משקל מחושב מ-e1RM (`calc1RM` ב-`archive-logic.js` ~1935, `_getExerciseE1RMPoints` ~2423; למיינים כבר יש `isCalc`/`baseRM` ב-`data.js`). ⚠ חייב לחול על שתי חזיתות הפיקרים (ui-main + Live) — הפיצ'ר הכבד ביותר.
- **B2. מדד אדהרנס שבועי + הזרקה ל-AI** 🟡 — אימונים שבוצעו מול מתוכננים + % ימי עמידה בחלבון/קלוריות; צד ה"בוצע" קיים (ארכיון עם date, תזונה יומית, `getTargetsForDate`, `_startOfWeek`); חסרה רק הגדרת "אימונים מתוכננים בשבוע" + פונקציית צירוף. הזרקה ל-`_buildCoachSummaryPrompt` ו-`buildSystemPrompt`. זהו "הדוח השבועי ההוליסטי" מהכיוונים הפתוחים.
- **B3. רצפים (Streaks) אוטומטיים** 🟢–🟡 — נגזרים מדאטה קיימת בלי צ'ק-אין ידני (חלבון 🔥 / אימונים / שקילות בוקר). אין כיום שום מנגנון streak; אם B2 קיים — B3 משתמש באותה שכבת צירוף.

### מסלול C — אכילה בחוץ (בהשראת MenuFit)
- **C1. סריקת תפריט מסעדה — "מה להזמין"** 🔴 — מצב סריקה רביעי: צילום תפריט → Gemini מדרג 2–3 מנות לפי היתרה היומית (קיימת ב-`_buildNutritionAIContext`) והמצב התזונתי; בחירה → Meal Builder. עיקרון מובנה מלקחי הביקורות על MenuFit: לנתח רק מה שבצילום, להציג כהערכה — לא להמציא מנות.
- **C2. הצעות התאמה למנה (Swaps)** 🟢 — שדה נוסף בסכמת C1 ("בקש בלי הרוטב"); אין טעם בלי C1.
