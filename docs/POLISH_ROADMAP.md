## P1 — טיפוגרפיה

**טוקנים (להוסיף ל-`:root`):**
```css
--fs-xs: .65rem;   /* מטא, תגיות, יחידות */
--fs-sm: .72rem;   /* טקסט משני, תתי-כותרות */
--fs-md: .8rem;    /* גוף */
--fs-lg: .9rem;    /* גוף מודגש, שורות רשימה ראשיות */
--fs-xl: 1.05rem;  /* כותרות סקשן */
--fs-2xl: 1.3rem;  /* כותרות מסך */
--fs-hero: 1.9rem; /* מספרי hero (בית, טבעת) — טיימר-מגה נשאר חופשי */
```

**מיפוי (borderline מוכרע ויזואלית per-component):**
| היום | יעד |
|------|-----|
| ≤0.68rem | --fs-xs |
| 0.7–0.78rem | --fs-sm |
| 0.8–0.85rem | --fs-md |
| 0.9–0.95rem | --fs-lg |
| 1–1.15rem | --fs-xl |
| 1.2–1.4rem | --fs-2xl |
| כל `em` | המרה ידנית ל-rem לפי הקשר (לחשב את הערך האפקטיבי בפועל!) |

**משקלים:** 900 = כותרות-ענק + מספרי hero · 700 = כותרות/הדגשה חזקה · 600 = הדגשת שורה ·
400 = גוף. **800 מתחסל** (→900 למספרים, →700 לטקסט). 200/300/500 → 400 או 600 לפי הקשר.

**tabular-nums:** utility `.num { font-variant-numeric: tabular-nums; }` + החלה ישירה על:
טיימרים (session strip, מנוחה, Live), פיקרים (משקל/חזרות/RIR), KPI (בית, Composition, אנליטיקה),
kcal ביומן (טבעת, שורות, preview), רשימת שקילות, ציר Y בגרפים.

**סיכון מרכזי:** איחוד 0.9→--fs-lg עלול לשבור שורות צפופות (session-log, טבלת TDEE). בדיקה פרטנית שם.

- **⏳ P1b — נדחה: המרת `em`.** ה-audit המקורי ספר ~14; בפועל **~120 מופעים**, כולם על טקסט בסקשנים

## P3 — Radius + ריווח

**Radius — סולם סופי:** `--r-xs: 8px` (חדש — צ'יפים/תגיות) · `--r-sm: 14px` · `--r-md: 20px` ·
`--r-lg: 28px` · pill 9999px · עיגול 50%.

| היום | יעד |
|------|-----|
| 6, 8px | --r-xs |
| 10, 12, 14px | --r-sm |
| 16px, 1rem, 18, 20px | --r-md |
| 2rem, 28px | --r-lg |
| 50px, 9999px | pill (9999px) |
| 50% | נשאר (עיגולים) |

**כלל קינון:** radius פנימי ≤ חיצוני − הפרש padding (ויזואלית: פנימי קטן מחיצוני).

**ריווח:** רשת מותרת {4, 8, 12, 16, 20, 24} + {2} למיקרו (אייקון↔טקסט צמוד).
מיפוי: 3→4 · 5→4 · 6→8 (זהירות: מרווח אייקונים צפוף — לשפוט ויזואלית, 6→4 אם 8 מנפח) ·
7→8 · 9→8 · 10→8 או 12 · 13,14→12 · padding באותה רשת.

## P4 — מושן

**טוקנים:**
```css
--dur-1: 120ms;  /* לחיצה, hover, טוגלים */
--dur-2: 200ms;  /* fades, החלפות תוכן */
--dur-3: 320ms;  /* sheets, מעברי מסך */
--ease: cubic-bezier(.2, 0, 0, 1);        /* standard decelerate */
--ease-spring: cubic-bezier(.34, 1.3, .4, 1); /* sheets בלבד */
```

- כל 130 ה-transitions → אחד משלושת המשכים + `--ease`.
- **מצב לחיצה אחיד:** כל אלמנט אינטראקטיבי (כפתורים, שורות רשימה, כרטיסים לחיצים, צ'יפים):
  `:active { transform: scale(.96-.98); }` עם `--dur-1`. לוודא שלא מתנגש ב-transform קיים
  (למשל sheets עם translateY — שם opacity בלבד).
- keyframes קיימים (18) נשארים; רק ליישר משכים בולטים לסולם.
- `prefers-reduced-motion: reduce` — קיים חלקית; לוודא כיסוי גלובלי ל-transitions החדשים.

**סיכון:** scale על אלמנט עם overflow/scroll עלול ליצור ריצוד ב-iOS — לבדוק על שורות רשימה ארוכות.

### P4 — כפי שבוצע (v16.96) + סטיות

- **טוקנים נוספו ל-`:root`** כמתוכנן (`--dur-1/2/3`, `--ease`, `--ease-spring`).
- **169 משכי transition מופו לטוקנים** (0.1/0.12/0.15→dur-1 · 0.18/0.2/0.22/0.25→dur-2 ·
  0.28/0.3→dur-3) + `var(--ease)` הוסף/החליף easing בכולם (160 הוספות, 9 החלפות).
- **אנימציות דאטה נשארו בכוונה מחוץ לסולם (8):** מילוי בארים/גרפים (`width 0.4–0.7s`),
  טבעות התקדמות (`stroke-dashoffset 0.9–1s`), `.bar all 0.5s` — האטה מכוונת לקריאת דאטה.
- **spring:** שני ה-bottom-sheets הראשיים (translateY(100%)) → `var(--dur-3) var(--ease-spring)`;
  `fdSlideUp` keyframe → spring גם כן.
- **10 אנימציות one-shot יושרו לסולם** (screenEnter ×2, fadeIn ×2, wqm-pop ×2, slideIn,
  stgPanelFade, fdSlideUp, fdItemIn). אנימציות infinite (פולסים, שימר, ספינר) — לא נגעו.
- **מצב לחיצה אחיד:** בלוק P4 בסוף הקובץ — `scale(0.98)` + transition מאוחד ב-dur-1 ל-28
  סלקטורים שהיה להם :active של background/opacity בלבד. **הוחרגו 6** שהבסיס שלהם משתמש
  ב-transform (range-card, archive-month-hd, km-select-card-pill, km-add-btn,
  km-add-to-cluster-btn, fd-entry) — מניעת התנגשות. כפתורי-אייקון עם opacity-press נשארו כמות שהם.
- **prefers-reduced-motion:** כיסוי גלובלי `*` כבר קיים (שורה ~1837) — לא נדרש שינוי.
- **לא טופל (חוב):** transitions שנכתבים מ-JS (סנאפ-בק של sheet ב-`workout-core.js`,
  טבעת ב-`food-logic.js`) — מחוץ לתחולת CSS-only.
- **לבדוק במכשיר:** ריצוד scale על שורות רשימה ארוכות ב-iOS (fd-food-row, archive-list-card, bl-row).
- ווריפיקציה: צילומי headless על 7 מסכים — 6 זהים ביט-לביט, ארכיון נבדל רק בפריים אנימציית כניסה.

## P5 — ליטושי פרימיום

1. **רשימות hairline:** שורות יומן מזון, רשימת שקילות, היסטוריית תזונה, ארכיון — קונטיינר אחד
   עם `border` + מפרידי `border-bottom: 1px solid var(--hairline)` פנימיים, במקום קופסה+רקע לכל שורה.
   (ראה מוקאפ.) **זהירות:** onclick על שורות נשאר זהה — שינוי CSS בלבד.
2. **גרבר אחיד ל-bottom sheets:** ידית 36×4px `--surface-4` בראש כל sheet (יש בחלק — לוודא בכולם).
3. **Empty states:** `emptyStateHtml()` הגלובלי קיים — לוודא שכל מסך ריק משתמש בו (יומן, ארכיון,
   אנליטיקה, שקילות, חיפוש מזון) עם אייקון+כותרת+פעולה באותה תבנית.
4. **Safe-area audit:** padding-bottom של תוכן גליל מעל tab-bar/session-strip בכל המסכים
   (כולל overlay יומן המזון) — `env(safe-area-inset-bottom)` עקבי.
5. **אופטיקה של אייקונים:** `Material Symbols` — לוודא `font-size` מסולם קבוע (1rem/1.2rem/1.5rem)
   ויישור אנכי אחיד (flex align-center) בכל שורות האייקון+טקסט.

   חודש אחד (r-lg), `.archive-list-card` שקוף עם מפרידים. ➌ **שורות יומן המזון (`.fd-entry`) כבר
   היו hairline** — לא נדרש שינוי. **סטייה:** שורות תוצאות חיפוש (`fd-food-row`) נשארו boxed —
   רשימה מעורבת (ai-row מקווקו, more-row) שאיחודה מסוכן; לא היו ברשימת הסעיף.
2. **גרבר:** כל ה-sheets כבר כללו ידית — בוצע איחוד סגנון בלבד: `.sheet-handle`/`.slog-handle`/
   `.ai-sheet-handle::after` → 36×4px, `var(--surface-4)`, radius pill.
3. **Empty states:** ההנחה במסמך הייתה ש-`emptyStateHtml()` קיים — **לא היה קיים; נוצר** גלובלי
   ב-`workout-core.js` + CSS `.empty-state` (עם `grid-column:1/-1` לתאימות גריד). הוחל ב-5 מוקדים:
   ארכיון ריק, שקילות ריק, תזונה-היום ריק, טאבי חיפוש מזון ריקים, אין-תוצאות-חיפוש.
   **סטייה:** בלי כפתורי פעולה (הפעולות ממילא צמודות למסך: capture-bar, חיפוש) — תת-כותרת מנחה
   במקום. טקסטים בתוך SVG של גרפים ("אין מספיק נתונים") נשארו — placeholder פנימי, לא מסך ריק.
4. **Safe-area:** נמצא ותוקן **באג env כפול** ב-`.freestyle-finish-float` (bottom חושב פעמיים);
   `.screen{padding-bottom:90px}` → `calc(90px + env(safe-area-inset-bottom,0px))` — כיסוי גורף
   לכל המסכים הגלילים. יתר 12 המופעים נבדקו — תקינים.
5. **אייקונים:** כבר מאוחדים על סולם הטיפוגרפיה מ-P1 (`--fs-xl`×22, `--fs-2xl`×8, גלובלי
   `line-height:1`) — תוקן מופע יחיד `1.1rem`→`--fs-xl`. הסולם בפועל הוא טוקני fs, לא
   1rem/1.2rem/1.5rem המילוליים מהתכנון — עקביות עדיפה על הערכים המדויקים.
- ווריפיקציה: צילומי headless כולל **הזרעת דאטה** (ארכיון 4 אימונים, 9 שקילות) — רשימות
  hairline + empty states אומתו ויזואלית; מסכים ללא שינוי נשארו זהים.
