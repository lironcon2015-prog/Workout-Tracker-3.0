# מתכון Shortcuts — סנכרון תזונה מ-Apple Health

מטרה: MyFitnessPal מזרים תזונה ל-Apple Health; קיצור דרך שולח את סיכום היום
(קלוריות + מאקרו) לגשר (`docs/health-nutrition-bridge.gs`) כמה פעמים ביום,
והאפליקציה מושכת משם אוטומטית בכל כניסה ובכל שעה עגולה כשהיא פתוחה.

> **חשוב:** MFP נשאר מקור האמת. ימי Health הם השלמה שוטפת בלבד — ייבוא MFP
> (מהגשר הקיים) דורס אותם ומוסיף פירוט ארוחות. שום דבר לא מתנגש.

> **דרישות מקדימות:** הגשר פרוס (ראה הראש של `health-nutrition-bridge.gs`) ·
> בהגדרות ה-PWA → "גשר תזונה Apple Health": URL + token ·
> MFP מחובר ל-Apple Health וכותב אליו תזונה (הגדרות MFP → Apps & Devices).

---

## הקיצור — "Health Sync" (חד-פעמי לבנות)

פתח את אפליקציית **קיצורי דרך** → ➕ קיצור חדש בשם `Health Sync`:

1. **Text** → הדבק את ה-URL של הגשר (`https://script.google.com/macros/s/…/exec`).
   שנה שם משתנה ל-`URL` (לחיצה ארוכה על הפעולה → Rename).
2. **Text** → ה-SECRET_TOKEN שהגדרת בסקריפט → משתנה `TOKEN`.
3. **Find Health Samples** (איתור דגימות בריאות):
   - Type: **Dietary Energy** (אנרגיה תזונתית)
   - Filter: **Start Date** → **is today** (התחלה היא היום)
4. **Calculate Statistics** → Operation: **Sum** על תוצאת שלב 3 → משתנה `CAL`.
   - אם מוצגת בחירת יחידה — בחר **kcal** (לא kJ!).
5. חזור על שלבים 3–4 עוד שלוש פעמים עבור:
   - **Protein** → משתנה `PROT`
   - **Carbohydrates** → משתנה `CARB`
   - **Total Fat** → משתנה `FAT`
   (היחידה לשלושתם: גרם)
6. **Format Date** → Date: **Current Date** · Format: **Custom** · תבנית: `yyyy-MM-dd`
   → משתנה `TODAY`.
7. **Get Contents of URL**:
   - URL: `URL`
   - Method: **POST** · Request Body: **JSON**
   - מבנה ה-JSON (הוסף שדות עם המשתנים):
     ```json
     {
       "token": TOKEN,
       "days": [
         { "date": TODAY, "calories": CAL, "protein": PROT, "carbs": CARB, "fat": FAT }
       ]
     }
     ```
     > את המערך `days` בונים ב-Shortcuts כך: שדה `days` מסוג **Array** → פריט
     > מסוג **Dictionary** → ובתוכו חמשת השדות. `date` כ-Text, השאר כ-Number.
8. (אופציונלי לבדיקה) **Show Result** — יציג את תשובת הגשר (`ok: true`).

**בדיקה ידנית:** הרץ את הקיצור → אמור להחזיר `{"ok":true,"stored":1,…}`.
אחר כך פתח את ה-PWA → מסך Composition → הנתונים של היום אמורים להופיע.

## האוטומציות — 4 ביום

באפליקציית קיצורי דרך → לשונית **אוטומציה** → ➕ → **אוטומציה אישית** → **שעה ביום**:

| # | שעה | תפקיד |
|---|------|--------|
| 1 | 10:00 | עדכון בוקר |
| 2 | 14:00 | עדכון צהריים |
| 3 | 18:00 | עדכון ערב |
| 4 | **23:45** | **סיכום היום הסופי** — האוטומציה החשובה |

בכל אוטומציה:
- חזרה: **יומי**.
- פעולה: **הפעל קיצור דרך** → `Health Sync`.
- **בטל את "שאל לפני הפעלה"** (Run Immediately) — אחרת תקבל התראת אישור בכל פעם.

> בהפעלה הראשונה iOS יבקש אישור גישה לנתוני הבריאות (קלוריות, חלבון, פחמימות,
> שומן) — אשר את כולם. זה קורה פעם אחת.

---

## מגבלות ידועות

- הקיצור שולח את **היום הנוכחי בלבד**. אם הטלפון כבוי בכל 4 הזמנים — היום
  יחסר עד ייבוא ה-MFP הבא (שמשלים הכל ממילא). נדיר ולא קריטי.
- ה-PWA מושך רק כשהוא פתוח (מגבלת iOS) — הנתונים פשוט מחכים בגשר.
- הגשר שומר ~120 ימים אחרונים; ההיסטוריה המלאה ממילא נשמרת באפליקציה ובענן.

## פתרון תקלות

| תופעה | סיבה/פתרון |
|--------|------------|
| `BAD_TOKEN` | ה-token בקיצור ≠ ה-token בסקריפט |
| `ok:true` אבל קלוריות 0 | MFP לא כותב ל-Health — בדוק MFP → Apps & Devices → Health |
| ערכים מוזרים (גדולים פי ~4) | היחידה kJ במקום kcal — תקן בשלב 4 |
| האוטומציה לא רצה לבד | "שאל לפני הפעלה" עדיין דלוק — כבה אותו |
