# TDEE — מתודולוגיית החישוב (GYMPRO ELITE)

מימוש: `computeTDEE()` ב-`bodylog-logic.js`. כרטיס: טאב תזונה ב-Composition. עוגן ל-AI: `_buildTdeeAIContext()` ב-`workout-core.js`.

## עיקרון
לא מסתמכים על נוסחה אחת (שגיאה ±200–500 קק"ל ליחיד). מריצים כמה שיטות, מעגנים על המדידה בפועל כשהיא קיימת, ומחזירים **טווח ביטחון** — לא מספר מדומה-מדויק.

## שיטות
| שיטה | קלט | מתי |
|------|-----|------|
| **מדידה (back-calc)** | תזונה יומית + מגמת משקל (≥14 ימים, ≥6 שקילות) | ה-gold standard — מעגן את ההערכה |
| **Katch-McArdle** | משקל + %שומן → LBM | תמיד כשיש שקילה עם %שומן |
| **Cunningham** | משקל + %שומן → LBM | כנ"ל |
| **Mifflin-St Jeor** | מין, גיל, גובה, משקל | כשפרופיל הגוף מלא |

- LBM = משקל × (1 − %שומן/100)
- BMR: Katch = 370 + 21.6·LBM · Cunningham = 500 + 22·LBM · Mifflin = 10·w + 6.25·h − 5·age + (זכר +5 / נקבה −161)
- TDEE תחזית = BMR × מכפיל פעילות (1.2–1.9)

## מדידה (back-calc)
```
TDEE = ממוצע_צריכה − שיפוע_משקל(ק"ג/יום) × 7700
```
- ממוצע צריכה: עם ניקוי ימי הזנה חסרה (`_cleanNutriOutliers`, <50% מהממוצע) כדי לא להטות כלפי מטה.
- שיפוע משקל: רגרסיה לינארית על השקילות בחלון (`_linRegSlope`), לא נקודה-מול-נקודה (רעש מים).
- חלון: 28 ימים אחרונים.

## Reconciliation + ביטחון
- יש מדידה → `best` = מדידה, CI ±7%, ביטחון "גבוה", אי-ודאות = דיוק הדיווח.
- אין מדידה → `best` = חציון התחזיות, CI ±10% (עם %שומן) / ±15% (בלי), אי-ודאות = מכפיל הפעילות.
- אם תחזיות סוטות >15% מהמדידה → דגל "דיווח לא עקבי".

## הערות
- 7700 קק"ל/ק"ג מניח שינוי שומן בעיקר; מספיק מדויק לתחזוקה.
- תת-דיווח עקבי מתבטל לצורך ההמלצה (הפלט באותן יחידות דיווח).
- מבוסס על סקיל `tdee-calculator` (Mifflin/Katch/Cunningham/Owen + back-calc + multipliers).
